## 1. 현황 진단 및 핵심 문제

### 1.1 페이지 구조 현황

```
TAX NAVI
├── [탭 1] 종합소득세 & 연말정산
│   ├── 입력 카드 (배우자 A + B + 부양가족) ← 폼 필드 약 40개
│   ├── 결과 카드 (종합 절세 통합 리포트)
│   ├── 연금저축/IRP 최적화 카드
│   └── 신용카드 황금비율 계산기 카드
├── [탭 2] 부가세/경비율
│   ├── 부가가치세 계산기
│   └── N잡러 경비율 비교기
├── [탭 3] 소득공제/세액공제
│   ├── 체육시설 이용료 공제
│   ├── 고향사랑기부제 최적화
│   ├── ISA 최적화 계산기
│   └── 건강보험료 시뮬레이터
└── [탭 4] 양도/증여/상속
    ├── 양도소득세 (부동산/주식)
    ├── 부동산 보유세
    ├── 간주임대료 계산기
    ├── 상속세 계산기
    ├── 혼인·출산 증여재산공제
    ├── 10년 주기 증여 플랜
    └── 자산 이전 절세 시뮬레이션
```

## 2. 모바일 UX 개선 계획

### 2.1 헤더 슬림화 (C/M-1)

**현재**: 로고 + 설명 텍스트 + 프라이버시 배지 + 테마 버튼 → 2줄 차지  
**개선**: 스크롤 다운 시 헤더를 compact 상태로 전환하는 **Shrinking Header** 패턴 적용

```
[TAX NAVI]  [🔒]  [🌙]
─────────── (이 이하로 스크롤 시 헤더 고정 + 슬림화) ───────────
```

**구현 방법**:
- 초기 헤더 높이: 64px → 스크롤 후: 44px (sticky)
- 프라이버시 배지 텍스트를 `tooltiptext`로 이동, 아이콘(🔒)만 노출
- 로고 설명 텍스트 모바일에서 숨김 (`display: none`)
- `IntersectionObserver`로 헤더 클래스 `header--compact` 자동 토글

### 2.2 탭 네비게이션 개선 (M-2)

**현재**: 4개 탭이 한 줄 → 375px에서 각 탭 80px 이하, 텍스트 잘림  
**개선**: 모바일 전용 하단 탭 바(Bottom Tab Bar) 도입

```
┌──────────────────────────────────────┐
│  (메인 콘텐츠 영역)                    │
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│ 🏠      🧾      🎯      🏢           │
│종합소득  부가세   공제   양도/증여      │
└──────────────────────────────────────┘
```

**구현 방법**:
- `position: fixed; bottom: 0;` Bottom Navigation Bar
- 아이콘(emoji) + 짧은 레이블(최대 4자) 구성
- 활성 탭: 언더라인 + accent color 강조
- Floating Result Bar와 z-index 충돌 방지: 결과 바는 bottom-nav 위에 stacking
- 데스크탑에서는 현재 상단 탭 유지 (`@media (max-width: 768px)` 조건부)

### 2.3 종합소득세 탭 입력 폼 대대적 재구성 (C-1, M-3, M-7)

**현재 상황 분석**:
- 배우자 A: 약 15개 필드 항상 펼쳐짐
- 배우자 B: 약 15개 필드 항상 펼쳐짐
- 부양가족 섹션: 무한 추가 가능

**개선안: Progressive Disclosure + Smart Collapse**

```
[배우자 A] [배우자 B]   ← 내부 탭으로 분리
─────────────────────
연봉: [___________]
소득유형: [근로소득 ▼]
카드사용: [___________]

▼ 금융소득 추가 (선택사항, 기본 닫힘)
▼ 연금저축/IRP (선택사항, 기본 닫힘)
▼ 주택자금 (선택사항, 기본 닫힘)
▼ 부양가족 (선택사항, 기본 닫힘)
```

**핵심 원칙**:
1. **Progressive Disclosure**: 필수 3~4개 필드만 기본 노출, 나머지는 접어서 숨김
2. **Single Column on Mobile**: 배우자 A와 B를 내부 탭으로 분리 (동시 노출 제거)
3. **Step-by-step**: 현재 Stepper UI를 진정한 Wizard로 전환 (동시 노출 → 순차 노출)

### 2.4 탭 4 내부 계산기 IA 재설계 (C-2)

**현재**: 7개 계산기가 단순 수직 스택  
**개선**: 내부 Segmented Control 도입

```
[탭 4: 양도/증여/상속]
┌─────────────────────────────────────┐
│ [양도소득] [보유세] [증여/상속]       │
│         (세그먼트 컨트롤)             │
└─────────────────────────────────────┘
  ↓ 선택에 따라 연관 계산기만 표시
```

**세그먼트 분류**:
- **양도소득**: 부동산/주식 양도소득세 + 자산이전 시뮬레이션
- **보유세**: 재산세+종부세 + 간주임대료
- **증여/상속**: 상속세 + 혼인출산공제 + 10년 주기 플랜

### 2.5 PDF 드롭존 접근성 개선 (M-5)

**현재**: 입력 카드 최상단에 항상 노출 (약 120px 고정 높이)  
**개선**: Disclosure Button 패턴으로 전환

```
[📄 홈택스 PDF로 자동입력 →]  ← 버튼 하나로 축소
     ↕ 클릭 시 드롭존 토글
```

- 기본 상태: 한 줄짜리 CTA 버튼
- 클릭 시: 드롭존 영역 펼침 (accordion)
- 사용자 세션에 기억: PDF 입력을 한 번 사용한 경우 기본 열림 상태 유지 (`localStorage`)

### 2.6 Floating Result Bar 개선 (N-4)

**현재**: "결과 보기" 버튼이 있지만 클릭 동작이 불명확  
**개선**: Bottom Sheet 패턴으로 교체

```
[플로팅 바]
합산 세액: 2,450,000원  [결과 보기 ↑]

  ↕ 클릭 시 Bottom Sheet 펼침

┌────────── (Bottom Sheet) ─────────┐
│ ↓ 닫기                             │
│ 🏆 종합 절세 통합 리포트            │
│ 배우자 A: 1,200,000원              │
│ 배우자 B: 1,250,000원              │
│ 최적화 절감: 350,000원 💚          │
│ ...전체 결과...                    │
└───────────────────────────────────┘
```

### 2.7 툴팁 모바일 대응 (M-6)

**현재**: CSS `::before` pseudo-element tooltip → 터치 화면에서 위치 이상  
**개선**: JS 기반 동적 tooltip 위치 계산 + Modal Tooltip on Touch

- 터치 기기: `?` 클릭 시 소형 Modal/Popover 노출 (화면 중앙 고정)
- 마우스 호버 기기: 현재 CSS tooltip 유지
- `ontouchstart` 여부로 분기 처리

## 3. 데스크탑 UX 개선 계획

### 3.1 2-Column 레이아웃 활용 극대화

**현재**: `grid-template-columns: 1.15fr 0.85fr` — 입력 좌측, 결과 우측  
**개선**: Sticky Result Card 도입

```
┌────────────────────┬──────────────────┐
│  [입력 폼]          │ [결과 카드] sticky│
│                    │ (스크롤해도 고정)  │
│  스크롤 가능        │                  │
│  ...               │                  │
└────────────────────┴──────────────────┘
```

**구현**:
```css
.result-card {
  position: sticky;
  top: 1.5rem;
  max-height: calc(100vh - 3rem);
  overflow-y: auto;
}
```

### 3.2 탭 4 데스크탑 레이아웃

**현재**: 7개 카드가 수직 나열 (단일 full-width)  
**개선**: 관련 계산기 그룹을 2~3열 그리드로 배치

```
┌────────────────────────────────────────┐
│         [양도/증여/상속]                │
├──────────────────┬─────────────────────┤
│  양도소득세       │  보유세 계산기        │
│  (부동산/주식)   │  (재산세+종부세)     │
├──────────────────┴─────────────────────┤
│      [Full Width: 상속/증여 계산기 그룹] │
└────────────────────────────────────────┘
```

### 3.3 키보드 접근성 및 Tab 순서

- 폼 필드간 Tab 이동 순서 논리적으로 재배열
- `tabindex` 속성 추가로 Stepper, Accordion 키보드 접근 가능
- `Escape` 키로 모달 닫기
- `Enter` 키로 계산 버튼 트리거

## 4. 공통 개선 계획

### 4.1 인라인 스타일 CSS 클래스화 (N-1)

**현재 상황**: `style=""` 인라인 스타일이 HTML 내 200개 이상 산재  
**개선**: 재사용 가능한 유틸리티 클래스 + 컴포넌트 클래스로 정리

```css
/* styles.css에 추가할 유틸리티 클래스 */
.section-group { background: rgba(255,255,255,0.02); padding: 8px; border-radius: 6px; margin-top: 10px; }
.section-group--primary { background: rgba(108,99,255,0.03); }
.section-group--secondary { background: rgba(0,212,170,0.03); }
.section-group__title { font-size: 0.75rem; font-weight: bold; margin-bottom: 6px; }
.result-box { padding: 1.5rem; border-radius: 12px; display: none; }
.result-box--info { background: rgba(56,189,248,0.05); }
.result-box--success { background: rgba(0,212,170,0.05); }
.result-box--purple { background: rgba(108,99,255,0.05); }
```

### 4.2 다크/라이트 모드 완전 대응

**현재 문제**: 인라인 `style="color: #fff"` 등이 모드 전환 시 대응 안 됨  
**개선**: CSS Custom Properties(변수) 확장

```css
:root {
  --surface-1: rgba(255,255,255,0.02);
  --surface-2: rgba(255,255,255,0.04);
  --surface-3: rgba(255,255,255,0.08);
  --surface-accent-a: rgba(108,99,255,0.05);
  --surface-accent-b: rgba(0,212,170,0.05);
}
body.light-mode {
  --surface-1: rgba(0,0,0,0.02);
  --surface-2: rgba(0,0,0,0.04);
  --surface-3: rgba(0,0,0,0.08);
  --surface-accent-a: rgba(108,99,255,0.06);
  --surface-accent-b: rgba(0,212,170,0.06);
}
```

### 4.3 카드 제목 간결화 (N-2)

**현재**: `🏟️ 체육시설 이용료 소득공제 (2025.7월 신설)` → 모바일 2줄  
**개선**: 짧은 Primary Title + 작은 Badge 조합

```html
<!-- Before -->
<h2 class="card-title">🏟️ 체육시설 이용료 소득공제 (2025.7월 신설)</h2>

<!-- After -->
<h2 class="card-title">
  🏟️ 체육시설 이용료 공제
  <span class="card-badge card-badge--new">2025.7 신설</span>
</h2>
```

```css
.card-badge {
  font-size: 0.62rem;
  padding: 2px 7px;
  border-radius: 4px;
  font-weight: 700;
  vertical-align: middle;
  margin-left: 8px;
}
.card-badge--new {
  background: rgba(0,212,170,0.15);
  color: var(--accent-secondary);
  border: 1px solid rgba(0,212,170,0.2);
}
.card-badge--updated {
  background: rgba(255,217,61,0.15);
  color: var(--accent-gold);
}
```

### 4.4 폼 입력 개선

#### 금액 입력 UX
- **천원 단위 구분**: 현재 구현됨 (유지)
- **한글 읽기 보조**: 현재 구현됨 (유지)
- **추가 개선**: 숫자 키패드 자동 노출 (`inputmode="decimal"` 통일)
- **Clear 버튼**: 0 이상 입력 시 × 버튼 노출 (간편 초기화)

#### 체크박스 / 라디오 터치 영역
- 최소 터치 영역 44×44px 보장 (현재 일부만 적용)
- `label` 래핑으로 클릭 영역 확장 (현재도 사용 중이나 일관성 부족)

### 4.5 로딩/빈 상태 일관성

**현재**: 계산기마다 `display:none` → 버튼 클릭 시 `display:block` 하드 전환  
**개선**: Fade-in 애니메이션 + 일관된 empty-state 컴포넌트

```css
.result-box {
  animation: fadeInUp 0.35s ease-out;
}
@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(8px); }
  to { opacity: 1; transform: translateY(0); }
}
```

## 5. 정보 아키텍처(IA) 재설계

### 5.1 현재 IA vs 제안 IA

| 현재 구조 | 제안 구조 | 이유 |
|---|---|---|
| 탭 1: 종합소득세 & 연말정산 | **탭 1: 내 세금 계산** | 사용자 멘탈 모델 (나 → 부부 순) |
| 탭 2: 부가세/경비율 | **탭 2: 사업자** | 명확한 타겟 그룹 분리 |
| 탭 3: 소득공제/세액공제 | **탭 3: 절세 전략** | 행동 지향적 라벨링 |
| 탭 4: 양도/증여/상속 | **탭 4: 자산 이전** | 사용자가 이해하기 쉬운 언어 |

### 5.2 탭 1 내부 계층 재설계

```
[내 세금 계산]
├── (A) 빠른 계산 — 핵심 필드 3~4개 (기본 노출)
│   ├── 연봉 / 소득유형 / 카드사용액
│   └── [🎯 바로 계산] 버튼
│
└── (B) 상세 계산 — 추가 공제 항목 (Progressive Disclosure)
    ├── [+ 배우자/부부 합산]
    ├── [+ 금융소득 (이자/배당/ISA)]
    ├── [+ 연금저축 / IRP]
    ├── [+ 주택자금]
    └── [+ 부양가족 추가]
```

### 5.3 탭 4 내부 계층 재설계

```
[자산 이전]
├── [양도] — 부동산/주식 양도소득세
│            + 자산이전(증여 후 매도) 시뮬레이션
├── [보유] — 재산세+종부세 보유세
│            + 간주임대료 (2026 신설)
└── [증여/상속] — 증여세 계산
                + 혼인·출산 특별공제
                + 10년 주기 증여 플랜
                + 상속세 계산
```
